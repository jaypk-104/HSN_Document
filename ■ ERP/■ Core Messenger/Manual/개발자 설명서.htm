[2] 서버 설치

 

1. 데이터베이스 설치 및 생성

 

   > 코어 메신저 서버는 기본적으로 MS-SQL을 데이터베이스로 사용합니다.

      메신저 서버를 설치하고하 하는 서버 컴퓨터 또는 별도의 서버 컴퓨터에 MS-SQL을 설치를 해야 합니다.

      (본 메신저 소스에는 MS-SQL은 포함되어 있지 않습니다. MS-SQL 이 없을경우, Express 버전을 설치하시면 됩니다.)

 

      설치가 되어 있다면 데이터베이스를 생성해야 하며, MS-SQL에서 Source\Core Messenger Database Script

      폴더 내부의 Core Messenger Database.sql 스크립트 파일을 실행하여 생성할 수 있습니다.

 

      아래는 MS-SQL 설치 예입니다.

      (여기서는 Microsoft SQL Server 2008 Express 로 진행했습니다.)

 

         

 

      우선 데이터베이스 만들기를 이용하여 Core Messenger 라는 데이터베이스를 생성합니다.

      다음으로 Core Messenger 데이터베이스를 선택 후 Source\Core Messenger Database Script 폴더 내부의

      Core Messenger Database.sql 파일을 열어서 실행합니다.

      아래와 같이 Core Messenger 데이터베이스에 10개의 테이블이 생성됩니다.

      보다 자세한 테이블 정보는 Manual 폴더의 코어 메신저 데이터 설명서.xls 파일을 참조하시면 됩니다.

 

         

 

2. 메신저 서버 컴파일 및 설치

 

   (1) Core Messenger Server 폴더 내부의 Core Messenger Server.sln 을 오픈 합니다.

   (2) 소스를 컴파일 후, Core Messenger Server.exe 파일을 설치하고자 하는 서버 컴퓨터에 복사합니다.

   (3) Command Prompt 창에서 서비스로 등록합니다.

        (예. C:\Messenger Server\Core Messenger Server.exe -install)

        (주의할 점은 윈도우 Vista 이상일 경우에는 관리자 권한으로 실행해야 합니다.)

   (4) Core Messenger Server Manager 폴더 내부의 Core Messenger Server Manager.sln 을 오픈 합니다.

   (5) 소스를 컴파일 후, Core Messenger Server Manager.exe 파일을 위의 (2)에서 설치한 서버 폴더에

        복사합니다.

   (6) Core Messenger Server Manager.exe 파일을 실행합니다.

 

        

 

   (7) [환경설정] 탭에서 위의 예와 같이, 데이터베이스 연결 문자열 및 파일 스토리지 폴더, 서비스 포트 번호를 입력 합니다.

        (데이터베이스 연결 문자열 예. server=127.0.0.1;database=Core Messenger;uid=sa;pwd=0000)

        (위의 설정은 예시이며, 해당 SQL 서버 주소 또는 인스턴스 명, 그리고 계정을 입력해야 합니다.)

 

         

 

   (8) [관리자] 탭에서 데이터 관리자 접속을 위한 관리자 아이디 및 비밀번호를 설정 후, [확인]을 클릭합니다.

   (9) 상단의 [제어] - [서비스 시작] 메뉴를 클릭하여 메신저 서버 서비스를 시작합니다.

        (서비스 시작전 방화벽에서 Core Messenger Server.exe 파일에 대한 방화벽 예외를 설정해 주어야 합니다.)

   (10) Core Messenger Server Manager.exe 를 종료합니다.

 

3. 메신저 업데이트 서버 컴파일 및 설치

 

   > 메신저 업데이트 서버는 보유한 웹서버가 없을 경우 사용할 수 있는 초소형 웹 서버 입니다.

      만일 Apache 또는 IIS 등의 웹서버를 보유하고 있다면 업데이트 서버를 설치할 필요는 없습니다.

      (업데이트 서버는 소규모의 메신저 사용에 적합하며 대량의 업데이트를 처리하지는 못합니다.)

 

   (1) Core Messenger Update Server 폴더 내부의 Core Messenger Update Server.sln 을 오픈 합니다.

   (2) 소스를 컴파일 후, Core Messenger Update Server.exe 파일을 설치하고자 하는 서버 컴퓨터에 복사합니다.

   (3) Command Prompt 창에서 서비스로 등록합니다.

        (예. C:\Messenger Update Server\Core Messenger Update Server.exe -install)

        (주의할점은 윈도우 Vista 이상일 경우에는 관리자 권한으로 실행해야 합니다.)

   (4) Core Messenger Update Server Manager 폴더 내부의

        Core Messenger Update Server Manager.sln 을 오픈 합니다.

   (5) 소스를 컴파일 후, Core Messenger Update Server Manager.exe 파일을 위의 (2)에서 설치한 서버 폴더에

        복사합니다.

   (6) Core Messenger Update Server Manager.exe 파일을 실행합니다.

 

        

 

   (7) 위의 예와 같이, 파일 스토리지 폴더 및 서비스 포트 번호를 입력 후, [확인]을 클릭합니다.

   (8) 상단의 [제어] - [서비스 시작] 메뉴를 클릭하여 메신저 서버 서비스를 시작합니다.

        (서비스 시작전 방화벽에서 Core Messenger Update Server.exe 파일에 대한 방화벽 예외를 설정해 주어야 합니다.)

   (9) Core Messenger Update Server Manager.exe 를 종료합니다.

 

 

[3] 사용자 정보 입력

 

1. 기본 코드 입력

 

    > 코어 메신저를 사용하기 위하여, 기본적으로 부서, 직급, 사용자 정보등을 입력하셔야 합니다.

       이러한 정보는 직접 데이터베이스에서 입력이 가능하지만, 코어 메신저 데이터 관리자를 사용하시면 편리합니다.

       코어 메신저 데이터 관리자는 데이터베이스에 접속하지 않고, 메신저 서버에 접속하여 데이터를 처리합니다.

 

   (1) Core Messenger Data Manager 폴더 내부의 Core Messenger Data Manager.sln 을 오픈 합니다.

   (2) 소스를 컴파일 후, Core Messenger Data Manager.exe 파일을 설치하고자 하는 컴퓨터에 복사합니다.

   (3) Core Messenger Data Manager.exe 파일을 실행합니다. 

 

        

 

   (4) [파일] - [프로그램 설정] 메뉴를 클릭합니다.

 

        

 

   (5) 프로그램 설정창에서 메신저 서버 주소 및 포트 번호를 입력 후 [확인] 버튼을 클릭합니다.

   (6) [파일] - [연결하기] 메뉴 또는 F2 키를 눌러 로그인 창을 엽니다. 

 

         

 

   (7) 메신저 서버 관리자에서 설정한 관리자 계정으로 로그인을 합니다.

   (8) 자세한 입력방법은 별도로 첨부한 관리자 설명서.htm 파일을 참조하세요.

 

 

[4] 클라이언트 설치

 

1. 메신저 클라이언트 컴파일 및 설치

 

    > 코어 메신저 클라이언트는 코어 메신저 자동 업데이트 소프트웨어를 같이 배포하셔야 합니다.

 

   (1) Core Messenger 폴더 내부의 Core Messenger.sln 을 오픈 합니다.

   (2) Module_Main.vb 파일에서 11번째 줄.

 

        '메신저 서버 아이피
        Public MessengerServerIP As String = "127.0.0.1"
        '메신저 서버 포트번호
        Public MessengerServerPortNo As Integer = 38000
        '홈페이지 주소
        Public CompanyWebSite As String = "www.coretive.com"

 

        부분을 원하시는 설정으로 변경합니다. 메신저 서버 아이피 및 포트번호는 메신저 서버가 설치된

        아이피 및 포트번호를 입력하시면 됩니다.

        (* 메신저 원격제어 포트번호는 서버로부터 수신되므로 설정할 필요가 없습니다.)

 

   (3) 소스를 컴파일 후, Core Messenger.exe 파일을 설치하고자 하는 사용자 컴퓨터에 복사합니다.

   (4) Core Messenger Update 폴더 내부의 Core Messenger Update.sln 을 오픈 합니다.

   (5) 소스를 컴파일 후, Core Messenger Update.exe 파일을 위의 (3)과 동일한 폴더에 복사합니다.

   (6) Core Messenger.exe 파일을 실행한 후, 부여된 계정으로 로그인 합니다.

 

 

[5] 설치 프로그램 만들기

 

1. 설치 프로그램 만들기

    > 코어 메신저는 기본적으로 닷넷 프레임워크 2.0 을 사용합니다.

       이는 WindowXP 이상 업데이트 설치시 자동으로 설치되므로 설치 프로그램 프로젝트에는 이를 고려하지 않았습니다.

       다시말해 설치 프로그램 프로젝트는 닷넷 프레임워크 2.0이 기본적으로 설치되어 있다는 가정하에, 해당 파일들

       설치 및 바탕화면 구성, 프로그램 구성을 처리합니다. 설치 프로그램 프로젝트는 코어 메신저 실행파일을 내부 리소스로

       포함하여 처리합니다.

       (첨부한 Install 폴더내부에는 미리 컴파일한 설치 프로그램이 포함되어 있습니다.)

 

       설치 프로그램을 통한 프로그램 설치시 자동으로 [시작] 메뉴 및 바탕화면에 바로가기 아이콘이 생성됩니다.

       메신저 서버 및 업데이트 서버의 경우, 자동으로 서비스 등록이 완료됩니다.

 

2. 코어 메신저 데이터 관리자 설치 프로그램 만들기

   (1) Core Messenger Data Manager Setup 폴더 내부의 Resource 폴더에, 컴파일한 Core Messenger Data Manager.exe

        파일을 덮어씁니다.

   (2) Core Messenger Data Manager Setup.sln 을 오픈 합니다.

   (3) 소스를 컴파일 합니다.

 

3. 코어 메신저 서버 설치 프로그램 만들기

   (1) Core Messenger Server Setup 폴더 내부의 Resource 폴더에, 컴파일한 Core Messenger Server.exe 파일과

        Core Messenger Server Manager.exe 파일을 덮어씁니다.

   (2) Core Messenger Server Setup.sln 을 오픈 합니다.

   (3) 소스를 컴파일 합니다.

 

4. 코어 메신저 설치 프로그램 만들기

   (1) Core Messenger Setup 폴더 내부의 Resource 폴더에, 컴파일한 Core Messenger.exe 파일을 덮어씁니다.

        (설치 프로그램 프로젝트에는 코어 메신저와 자동 업데이트 소프트웨어가 같이 포함됩니다.

        만일 자동 업데이트 소프트웨어까지 수정하게 된다면, 해당 파일도 덮어쓰시면 됩니다.)

   (2) Core Messenger Setup.sln 을 오픈 합니다.

   (3) 소스를 컴파일 합니다.

 

5. 코어 메신저 업데이트 서버 설치 프로그램 만들기

   (1) Core Messenger Update Server Setup 폴더 내부의 Resource 폴더에, 컴파일한 Core Messenger Update Server.exe

        파일과 Core Messenger Update Server Manager.exe 파일을 덮어씁니다.

   (2) Core Messenger Update Server Setup.sln 을 오픈 합니다.

   (3) 소스를 컴파일 합니다.

 

 

[6] 코어 메신저 업데이트 처리

 

1. 코어 메신저 버전 수정

    > 코어 메신저 업데이트는 내부 버전 체크 비교 방식을 사용합니다. 수정(또는 추가)된 코어 메신저 소스에서

       해당 버전을 보다 높은 숫자로 변경 후 컴파일 하고, 이 파일을 업데이트 서버에 복사 및 업데이트 정보 설정

       과정을 통해서 진행됩니다.    

 

   (1) Core Messenger 폴더 내부의 Core Messenger.sln 을 오픈 합니다.

   (2) 기능 수정 또는 추가를 처리합니다.

   (3) 솔루션 탐색기에서 My Project를 더블 클릭 합니다.

   (4) 응용 프로그램 탭에서 [어셈블리 정보]를 클릭 합니다.

 

        

 

   (5) 어셈블리 버전과 파일 버전을 동일하게, 기존 버전보다는 높게 설정합니다.

        (예를 들면 기존 버전이 1.0.0.1 이면 수정은 1.0.0.2 식으로 설정하면 됩니다.)

   (6) 소스를 컴파일 합니다.

   (7) 코어 메신저 업데이트 서버 관리자를 실행합니다.

   (8) 파일 스토리지 폴더란에 설정된 경로에, 위에서 컴파일한 Core Messenger.exe 파일을 복사합니다.

   (9) 코어 메신저 업데이트 서버 관리자를 닫습니다.

   (10) 코어 메신저 서버 관리자를 실행합니다.

   (11) 업데이트 탭에서 메신저 업데이트 URL 란에 코어 메신저 업데이트 서버의 주소 및 경로와 파일명을 입력합니다.

          (예를 들면 업데이트 서버 컴퓨터의 아이피가 192.168.0.1 이면, http://192.168.0.1/Core Messenger.exe 로

           입력하면 됩니다.)


         * 여기서 중요한점은 포트번호 설정입니다. 업데이트 서버의 서비스 포트번호가 38002 번 이라면, 업데이트 URL을

             http://192.168.0.1:38002/Core Messenger.exe 이런식으로 수정하시면 됩니다. 실제로 웹브라우저의 주소창에서

            위의 URL을 입력했을 경우, 파일 다운로드창이 표시되면 이상이 없는 경우 입니다.

 

   (12) 메신저 업데이트 버전 란에는 위에서 수정한 코어 메신저의 버전을 입력합니다.

          (위의 예를 들면 1.0.0.2 로 입력하면 됩니다.)

   (13) 확인을 누른 후 창을 닫습니다.

   (14) 이후 코어 메신저를 종료 후 새로 실행하게 되면, 로그인 시 서버에 설정된 버전과 비교하게 되며, 새 버전이 있을경우

          코어 메신저에 포함된 자동 업데이트 소프트웨어가 업데이트를 진행하게 됩니다. 만일 네트워크 문제 또는 여러 문제로

          업데이트가 불가능 할경우, 무한 업데이트 처리를 방지하기 위해서 코어 메신저를 종료 후 재시작 하기 전까지는

          다시 업데이트를 처리하지 않습니다.
